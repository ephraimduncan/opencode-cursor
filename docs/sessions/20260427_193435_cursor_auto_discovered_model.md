# Cursor auto discovered model mapping

Date: 2026-04-27
Repo: `C:\NEW PRG\opencode-cursor`

## Problem

`explore` subagent calls routed through `cursor/auto` failed before doing any search with:

- `[Error: Connect error invalid_argument: Model details are required]`

Earlier work made `cursor/auto` omit `AgentRunRequest.modelDetails`, assuming Cursor would resolve a default model. Live subagent tests proved that assumption wrong for the private `agent.v1.AgentService/Run` path.

## Research summary

- Oracle review and external OSS research agreed that private Cursor `AgentService/Run` requires `modelDetails`.
- Public Cursor Cloud Agents REST docs support `model: "default"` or omitted model, but that behavior does not apply to the private ConnectRPC protobuf path.
- OSS implementations using `AgentService/Run` consistently send `modelDetails` with an explicit model ID.
- `requestedModel` examples still include `modelDetails`; there is no evidence that `requestedModel` alone replaces it.

## Changes

- `src/proxy.ts`
  - Added `resolveCursorRunModelId()`.
  - Kept `auto` in the OpenAI-facing `/v1/models` list.
  - Mapped OpenAI-facing `model: "auto"` to the first discovered non-auto Cursor model before building `ModelDetails`.
  - Explicit model requests still pass through unchanged.

- `test/smoke.ts`
  - Added fake Cursor backend observation of `AgentService/Run` protobuf payloads.
  - Added a smoke test proving:
    - `model: "auto"` includes `modelDetails`.
    - auto resolves to the first discovered Cursor model (`composer-2` in the test fixture).
    - explicit `composer-2` still forwards unchanged.

- `dist/proxy.js`
  - Regenerated by `bun run build`.
